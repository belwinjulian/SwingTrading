# Momentum Swing Screener

A long-only, end-of-day swing-trading screener that scans the Russell 1000 every evening and produces a ranked list of stocks worth buying tomorrow. Each pick declares which playbook it fits — Qullamaggie continuation flag, Minervini VCP, or leader-hold — and surfaces a concrete entry, stop, and position size for that playbook.

**Status:** Phase 1 — repository scaffolding. No data fetching, indicators, or backtests yet.

## Setup

Requires Python 3.11 and [uv](https://docs.astral.sh/uv/).

```bash
uv sync --extra dev
cp .env.example .env       # then fill in API keys
uvx pre-commit install
```

All workflows run through `make`:

```bash
make help        # list available targets
make setup       # uv sync + pre-commit install
make data        # refresh universe + ohlcv + macro + fundamentals (stub in Phase 1)
make rank        # compute composite scores (stub in Phase 1)
make report      # render the daily markdown report (stub in Phase 1)
make backtest    # run the walk-forward backtest (stub in Phase 1)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer-machine ↔ git-remote | Source code and lockfile cross here; secrets must NEVER cross |
| pyproject.toml ↔ external-pypi | Dependency resolution pulls third-party code; supply-chain risk |
| .env ↔ git-remote | API keys live in .env locally; must NEVER be committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | pyproject.toml dependency line | mitigate | Compat-style version pins (`>=X,<X+1`) per D-02; `uv.lock` committed for byte-reproducible installs (D-02); Task 1 forbids the `pandas-ta` PyPI package whose maintainer changed (STACK.md "What NOT to Use") |
| T-01-02 | Tampering | uv.lock | mitigate | CI uses `uv sync --frozen --extra dev` (D-09 in Plan 05); any drift between pyproject.toml and uv.lock fails the build |
| T-01-03 | Information Disclosure | .env file | mitigate | `.env` listed in `.gitignore` (Task 3); `.env.example` is the committed template; pre-commit hook in Plan 05 runs ruff/mypy but secret-scanning is deferred (acceptable for v1 — no real keys yet) |
| T-01-04 | Information Disclosure | EDGAR_IDENTITY (personal email) | accept | Personal email in EDGAR_IDENTITY is required by SEC policy and ships in `.env.example` as a placeholder only; user fills in real value in `.env` which is gitignored. Per PITFALLS.md "Security Mistakes" §"EDGAR set_identity()": acceptable for personal-trading repo, dedicated alias optional |
| T-01-05 | Denial of Service | uv sync supply chain | accept | Free PyPI mirror; no commercial SLA. Compat-style pins limit blast radius. Belwin's $0 budget rules out paid mirror. Recovery: pin to last-known-good versions in pyproject.toml |
| T-01-06 | Elevation of Privilege | hatchling build backend | accept | hatchling is Astral's modern uv-native default (D-04); Apache-2.0 licensed; widely adopted. No alternative within $0 budget. |

</threat_model>
